Sistema de Grid
Grid: Desktop 18 colunas (>992px), Tablet 12 colunas (768-992px), Mobile 6 colunas (<768px). Classe: D{n}T{n}M{n}. Container de linha: .linhaDoGrid.
Configuracoes do GRID:
- Desktop - 18 Colunas
- Tablet - 12 Colunas
- Mobile - 6 Colunas
Estrutura base do grid responsivo:
  • linhaDoGrid e o container externo de UMA linha do grid. Define a largura total disponivel (calc(100vw - scrollbar)) e ja inclui as margens/gutters do breakpoint atual. NUNCA pode ser aninhado dentro de outro linhaDoGrid.
  • panel aplica caixa com borda arredondada, padding interno e fundo do componente. Use em qualquer filho do grid que precisar de visual de "cartao".
  • D{n}T{n}M{n} define quantas colunas o elemento ocupa em cada breakpoint: D=Desktop (18 cols), T=Tablet (12 cols), M=Mobile (6 cols). Aqui D18T12M6 ocupa 100% em todos os breakpoints.

18 colunas em todas as resolucoes:

Ocultar colunas por viewport:
  • Valor 0 em qualquer dimensao significa display: none naquele breakpoint. Use para esconder elementos em telas menores.
  • D1T1M0: visivel no Desktop e Tablet, oculto no Mobile.

Colunas com largura multipla:
  • Aumente o numero apos D, T ou M para ocupar mais colunas. D2T2M2 = 2 colunas em todos os breakpoints.
  • No Desktop, 9 elementos D2 ocupam 18/2 = 9 espacos (cabe tudo em 1 linha). No Tablet, 6 elementos T2 = 12/2 (1 linha de 6, depois quebra). No Mobile, 3 elementos M2 = 6/2 (3 por linha).
  • Combine com 0 para ocultar progressivamente: D2T2M0 esconde no mobile, D2T0M0 esconde no tablet e mobile.

Deslocamento (offset) na 1a coluna:
  • desloc-D{n}T{n}M{n} empurra o elemento N colunas para a direita, criando um espaco vazio antes dele. Funciona como margin-left de N colunas.
  • Aplique como classe ADICIONAL ao mesmo elemento que ja tem D{n}T{n}M{n}.
  • Como o offset ocupa espaco no grid, e preciso REMOVER um elemento do final para nao quebrar a linha. Por isso o ultimo aqui usa D0T0M0 (display none em todos breakpoints).
  • Cada breakpoint tem seu offset independente: desloc-D2T1M0 empurra 2 cols no Desktop, 1 col no Tablet e nada no Mobile.

Offset aplicado em qualquer posicao:
  • O desloc-D{n}T{n}M{n} nao precisa ser no primeiro elemento. Aplique em qualquer filho do grid e o framework vai inserir o espaco vazio ANTES dele.
  • Util para criar gaps entre grupos de elementos sem precisar de divs intermediarias vazias.
  • Mesma regra de compensacao: ao usar offset, remova um elemento equivalente do final para a linha nao quebrar (aqui o 18 com D0T0M0).

Classes item-1 a item-30 reordenam os elementos via CSS flex order, sem alterar a ordem no HTML.

Reordenacao visual sem alterar o HTML:
  • item-N (de item-1 ate item-30) define a posicao visual de exibicao via flex order. O HTML mantem sua ordem original (importante para acessibilidade e SEO) mas o usuario ve em outra ordem.

Containers de coluna e linha (sub-layouts):
  • containerEmColuna empilha seus filhos verticalmente. Recebe a classe de grid D{n}T{n}M{n} para definir sua propria largura no grid pai.
  • containerEmLinha distribui filhos horizontalmente como sub-grid (com margens negativas compensatorias). Pode ser aninhado dentro de containerEmColuna.
  • D-nowrap, T-nowrap, M-nowrap impedem quebra de linha naquele breakpoint. Quando o conteudo nao cabe, gera scroll horizontal interno.
  • nowrap aplica nowrap em todos os breakpoints de uma vez.
  • Use estas classes para criar sub-layouts dentro de uma linhaDoGrid sem precisar aninhar outra linhaDoGrid (proibido).

Mistura de filhos diretos com containerEmColuna:
  • Dentro de uma mesma linhaDoGrid voce pode misturar livremente filhos diretos com classes D{n}T{n}M{n} e subcontainers containerEmColuna que tambem recebem classe de coluna.
  • O containerEmColuna ocupa o mesmo espaco que ocuparia um panel simples, mas internamente empilha varios elementos verticalmente — util para criar grupos de cards alinhados.
  • NUNCA aninhar linhaDoGrid dentro de outro linhaDoGrid — isso quebra o calculo de largura e gera scroll lateral. Para sub-grids use containerEmColuna ou containerEmLinha.

Formularios

Campos com icone — containerItemFormComIcone
Inputs com icone interno:
  • containerItemFormComIcone e o wrapper que cria o contexto position: relative para o icone ser posicionado em cima do input. Aplica padding-left no input para o texto nao sobrepor o icone.

Formulario completo: passos, titulo, balloons:
  • passos e passoCompleto / passoFuturo: indicador visual de etapas do formulario, com check/numeracao. O passo atual (sem modificador) recebe destaque automatico.
  • tituloForm: barra de titulo com gradiente da cor primaria, padronizada. Use arredondadaSuperior em conjunto com arredondadaInferior no panel abaixo para fechar visualmente o conjunto.
  • panel containerEmColuna: agrupa varios campos verticalmente sob o titulo.
  • containerHelpIcon: wrapper externo que alinha o input + um botao de ajuda (help-icon) ao lado direito.
  • containerItemFormComIcone: position: relative que serve de referencia tanto para o icone quanto para o balao com position: absolute.
  • balloonTrigger: marcador para o JS escutar o click e abrir/fechar o balao via classe balloonStatus-Ativo.
  • O balao e filho direto de containerItemFormComIcone (ou irmao do trigger). O balao DEVE ter as classes balloonPosition-All-{Left|Right|Top|Bottom|TopLeft|TopRight|BottomLeft|BottomRight}, balloonType-{Help|Atencao|Erro} e D{n}T{n}M{n} (define a largura).

Message Balloons
Tipos: balloonType-Help, balloonType-Atencao, balloonType-Erro. Posicoes: balloonPosition-All-{Left|Right|Top|Bottom|TopLeft|TopRight|BottomLeft|BottomRight}. Ativados via clique com .balloonTrigger (JS).
3 tipos de balao — clique nos campos para abrir/fechar
Mensagem de ajuda em fundo cinza com borda preta clara. Usado para dicas contextuais.
Mensagem de atencao em fundo laranja (@CorAtencao). Usado para avisos.
Mensagem de erro em fundo vermelho (@CorNegacao). Usado para erros e bloqueios.
3 tipos de balao com cores semanticas:
  • balloonType-Help: fundo cinza claro com borda preta, para dicas contextuais e instrucoes neutras.
  • balloonType-Atencao: fundo laranja (@CorAtencao) para alertas e avisos importantes ao usuario.
  • balloonType-Erro: fundo vermelho (@CorNegacao) para mensagens de erro de validacao e bloqueios.
  • Cada balao usa um triangulo CSS (via :before/:after com border colorido) que se ajusta automaticamente conforme a balloonPosition-All-* para apontar para o input/icone de origem.
  • O readonly nos inputs aqui e so para a demo (impede o teclado abrir). Em uso real, omita.
<div class="linhaDoGrid"> <div class="containerHelpIcon D6T4M2"> <div class="containerItemFormComIcone"> <input type="text" class="itemForm input balloonTrigger" placeholder="Click me" /> <span data-icon="info"></span> <!-- Tipo: balloonType-Help | balloonType-Atencao | balloonType-Erro --> <div class="balloonPosition-All-Bottom balloonType-Help D3T2M1"> Texto do balao </div> </div> </div> </div>

8 posicoes possiveis do balao:
  • balloonPosition-All-{Posicao} define a direcao em que o balao aparece em relacao ao trigger. As 8 posicoes sao: Top, Bottom, Left, Right, TopLeft, TopRight, BottomLeft, BottomRight.
  • All aplica em todos os breakpoints. Para alterar por breakpoint use balloonPosition-Desktop-*, balloonPosition-Tablet-* ou balloonPosition-Mobile-*.
  • help-icon: botao circular pequeno (semelhante ao "?" de ajuda) que ja vem com position: relative e padding correto para servir de ancora ao balao.
  • O balao deve ser filho direto do elemento ancora (help-icon ou containerItemFormComIcone), nunca solto na pagina.
  • O triangulo aponta sempre para a borda do trigger oposta a direcao em que o balao aparece (ex: Bottom -> triangulo no topo).

Estrutura completa da navbar:
  • linhaDoGrid navbar: combina o container do grid com a classe especial navbar, que aplica o gradiente de fundo, sombra e altura padronizada da barra.
  • hamburgerCheckbox + hamburgerDisplayButton + hamburgerDisplayLines: menu hamburguer 100% CSS (sem JS), via input checkbox oculto. Aparece automaticamente apenas em breakpoints menores.
  • menusContainer: wrapper que agrupa menuPrincipal e menuSecundario e controla a visibilidade quando o hamburger e clicado.
  • menu menuPrincipal: menu da esquerda com itens principais. menu menuSecundario: menu da direita (geralmente login/perfil), recebe cor secundaria.
  • menuItem: cada item do menu. Modificadores: inativo (cinza, sem cursor pointer), paginaAtual (destaque visual da pagina atual).
  • iconMenu data-icon="nome": span dentro do <a> que renderiza um icone SVG inline ao lado do texto do item (puxado de /icones/{nome}.svg).
  • subMenu: dropdown que aparece ao passar o mouse no menuItem (Desktop) ou ao tocar (Tablet/Mobile). Contem subMenuItem (item normal) e subMenuItem titulo (cabecalho de secao, nao clicavel).
  • containerExternoBotoesSubmenu + containerInternoBotoesSubmenu: layout para botoes lado a lado dentro do submenu.
  • containerbtFacebook / containerbtGoogle: estilos prontos de botoes de login social com cor de marca correta.

Textos por Breakpoint
Classes .spanDesktop, .spanTablet, .spanMobile exibem conteudo distinto por resolucao.
Voce esta vendo: DESKTOP (>992px) TABLET (768-992px) MOBILE (<768px)
Conteudo de texto diferente por breakpoint:
  • spanDesktop: visivel apenas no Desktop (>992px), oculto via display: none nos demais.
  • spanTablet: visivel apenas no Tablet (768-992px).
  • spanMobile: visivel apenas no Mobile (<768px).
  • Util quando o texto precisa ser DIFERENTE por viewport (nao apenas o tamanho da fonte). Ex: "Adicionar ao carrinho" no desktop vs "+ Carrinho" no mobile.
  • Os 3 spans coexistem no HTML — apenas um e visivel por vez. Para SEO, todos os textos sao indexados.

Icones SVG
80 icones em /icones/{nome}.svg. Uso no framework: inline via <!--#include virtual="~/App_Themes/icones/nome.svg"--> ou <img src="/icones/nome.svg">.
Biblioteca de icones SVG do framework:
  • Todos os icones SVG ficam em /icones/{nome}.svg (cerca de 80 disponiveis). Cada arquivo e um SVG monocromatico que respeita as cores aplicadas via CSS (fill, stroke).
  • Uso inline (recomendado): <span data-icon="user"></span> mais um script global que faz fetch e injeta o SVG. Permite estilizar via CSS (mudar cor, tamanho, hover).
  • Uso direto via <img>: <img src="/icones/user.svg">. Mais simples, mas o SVG nao herda currentColor nem responde a CSS externo.
  • O grid acima e populado em runtime pelo endpoint GET /api/icons do servidor — em uso normal voce escolhe os icones que quer e referencia individualmente.
<!-- Grid populado dinamicamente via JS a partir de /api/icons --> <div id="iconsGrid" class="linhaDoGrid" style="flex-wrap:wrap;gap:4px;padding:0 6px"></div> <script> fetch('/api/icons') .then(function(r) { return r.json(); }) .then(function(icons) { var grid = document.getElementById('iconsGrid'); icons.forEach(function(icon) { var div = document.createElement('div'); div.innerHTML = '<img src="/icones/' + icon + '.svg" /><small>' + icon + '</small>'; grid.appendChild(div); }); }); </script> <!-- Uso individual num projeto: --> <img src="/icones/user.svg" alt="user" /> <!-- Ou inline (estilizavel via CSS fill/stroke): --> <span data-icon="user"></span>
